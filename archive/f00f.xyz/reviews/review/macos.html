macOS is an operating system made by Apple. It's the second most popular OS in the entire world, grabbing a nice 10-20% of the desktop marketshare.

Packages

ALL OF THEM. ALL. NONE LEFT BEHIND.

Not only is almost every GAHNOO/LEENCUX package ported and is downloadable via MacPorts or brew, you also get a shitload of professional software. This includes, but is not limited to

  • All of Adobe's line up. Winpajeets will hate this - these tools are native to macOS and work a hell of a lot better here, Windows is a shitty port.
  • A proper fucking Chrome build. I'm sorry but the Linux port of Chrom{e,ium} is a pile of RAM-leaking shit. I cannot help but notice the seconds of lag and near-gigabytes of RAM usage in every Electron app. Shit's sooooo much smoother on macOS (same goes for Windows)
  • All Apple's professional software - like Final Cut Pro or Logic. Although many buttmad lincucks love to shit on these tools saying that "everyone that uses them hates them", that is simply not true. They may not be the best if you've already mastered something like Premiere Pro, but you cannot deny their popularity in the professional stage.

At this point I'll have to say - macOS has all the software. Everything except the boomer Windows tools like procmon.exe and mIRC and foobar2000 or other shit like that.

On it being up-to-date, macOS has no package manager, which means that every application checks for a newer version itself. Linux users foam at the mouth hearing this approach, but let me tell you why it's superior. It's a startup check. It takes no time and you'll get updates the fastest. No bullshit if the repository breaks down or your package manager breaks down or you just being lazy and not updating for a little while.

And by the way, I use brew, which has pretty up-to-date shit, so no problem there.

Installation

Well... fuck.

There are two ways to install macOS

The right way

This is if you own a Mac, you'll just pop out the .app on an USB drive, click through a few buttons and you'll have a fully functioning system in the matter of minutes. Or you may have an older Mac which doesn't support the version of macOS you want to install, then you're just going to get some patches from dosdude1 and follow the extremely detailed and simple instructions to get a working system.

The... um... ugly way

And if you're too poor to own a Mac, you'll have to go through the fucking hell to get it to work. macOS completely hates many pieces of hardware and most just end up picking a list of parts before building a PC specifically for hackintoshing. If you're like me and own a laptop, well, you must be as lucky to have a hackintoshing community for that specific laptop. I own an X220 which has a pretty large hackintoshing community consisting of around 5 members. This means I just dragged a folder full of kexts and patches to get my install working. Other laptops, you may not even get that shit working. Ever. I know this when I tried to hackintosh a Packard Bell. The ATI 5740M card is COMPLTELY unhackintoshable. No way to get it working. Just get a different laptop at that point. Yeah, this is ugly, and frankly, the worst part of OS X. I am glad that there exists a community making an effort, I'm mad that it has to exist.

Usability

Advanced use

I don't know how advanced of an user I am, but I can do some complex shit sometimes, and macOS pretty much held up nicely all the time. As I said, all the Garbage/Linshit tools are ported over and macOS uses an UNIX shell, which is really nice.

What you'll notice though is that macOS is UNIX-standard compliant somewhat, but implements some of its own stuff info the mix, here are some gripes I have about it (but don't take these as bad things)

  • The /home (or /usr/home) directory is /Users. I don't really understand this. Clarity? Yes. Necesarry? No.
  • It automounts shit to /Volumes/[Volume name]. Yes. This a thousand times yes. I mean I didn't mind doing mount /dev/sdb1 /mnt all that much all these years, but having automounted stuff is quite convenient. But the way Linux DEs do it is absolutely retarded. /media/[username]/random-shit/*. Really? WHY? This is terribly inane! What the fuck is the point of this random bullshit? Why not use the volume name? I literally had to open the file manager to find which volume that was in there was what. And I had to keep that file manager open because I would keep forgeting what the fuck a918-1kna and ij1k-997c meant. Ridicilous.

Simple use

This is where macOS shines. Unlike Linux where everything is made needlessly complicated and now every DE is trying to dumb it down but is failing, macOS has everything a simple user would ever need in a simple and easy-to-understand GUI. Now I don't want to make this sound like a system made for idiots like Windows. macOS strikes a balance of making simple things simple and letting advanced users do advanced things. Even as a pretty good computer man, I still like using the simple GUIs that macOS gives, unlike where in Linux it was the terminal or nothing.
